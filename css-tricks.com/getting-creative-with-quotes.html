Getting Creative With Quotes

First, a quote-unquote ‚Äúrecap.‚Äù

There are no limitations on how quotations can be styled. Block and pull quotes can both be eye-catching design elements, but they convey different messages. While a block quote is typically inside the content flow, a pull quote (sometimes called a callout) is extracted from the text to form a separate element.

The proper HTML for marking up a block quote depends on its contents. My design for Patty Meltt includes concert reviews, which contain the reviewer‚Äôs name:

  <p>"Patty Meltt‚Äôs powerful ballads and toe-tapping anthems had the audience singing along all night."</p>

</blockquote>

Here, the footer contains information about the source or author of the parent element. This makes it a good fit for attributions inside a blockquote, where it indicates who wrote it. But what about cite?

For years, I used the cite element to mark up attributions. It‚Äôs one of those sneaky bits of HTML that felt intuitive until I read the spec and went, ‚ÄúDagnabbit!‚Äù because cite isn‚Äôt meant to label people. Instead, it should be used for:

So, in that example, footer marks up the attribution, and cite points to the title of the publication, not the person writing it. This gives the markup a semantic boost and helps people who use screen readers.

Styling with personality

Out-of-the-box, browsers do very little to style blockquotes, except for adding inline margins. You could add some simple blockquote styling, but with just a little more style, you can transform them into expressive design elements that reflect a brand‚Äôs personality and voice.

For Patty Meltt‚Äôs design, I wanted her quotes to feel confident, loud, and a little over the top.

Borders

A simple border, used well, can make block and pull quotes stand out and anchor them into a layout. A border on the left or top separates a block quote from surrounding content, helping a reader recognise it as a different voice from the main narrative.

In magazines and newspapers, block quotes punctuate content blocks and are frequently styled to contrast with the surrounding text. A full-width, bordered block quote encourages a reader to pause for a moment.

It‚Äôs a simple, yet effective, way to focus someone‚Äôs attention on a message. A thin border feels quiet and understated:

  padding-inline: 1.5rem;
  border-inline-start: 1px solid #98838e;
  border-inline-end: 1px solid #98838e;
}

This may suit some brands, but that‚Äôs not a style which reflects Patty‚Äôs personality. Whereas a bolder, thicker border feels more confident, like it has something important to say:

  padding-inline: 1.5rem;
  border-inline-start: 5px solid #98838e;
  border-inline-end: 5px solid #98838e;
}

Those borders needn‚Äôt always fill the full height or width of a blockquote, so instead of using the border property, use ::before and ::after pseudo-elements to add faux borders at any size:

  display: flex;
  flex-direction: column;
  align-items: center;
}

  content: "";
  display: block;
  width: 80px;
  height: 5px;
  background-color: #98838e;
}

You could even animate those faux borders using keyframe animations or simple transitions to increase their width when someone interacts with the quotation:

  content: "";
  display: block;
  width: 80px;
  height: 5px;
  background-color: #98838e;

  width: 100%;
}

Quote marks

Before choosing how to style your quote marks, consider whether you need them at all. Technically, an HTML blockquote implies its content is a quotation. So, from an accessibility and semantic standpoint, quote marks aren‚Äôt required because screen readers and search engines will recognise a blockquote. However, quote marks can visually emphasise quoted content and add interest and personality to a design.

Are both opening and closing marks always needed? Possibly, when a design needs a traditional feel, or a quotation appears in a passage of running text:

blockquote {
  position: relative;
  padding-inline: 64px;
}
  
blockquote img:first-of-type,
blockquote img:last-of-type {
  position: absolute;
}

blockquote img:first-of-type {
  top: 0;
  left: 0;
}

blockquote img:last-of-type {
  right: 0;
  bottom: 0;
}

Or, to give a design an editorial feel, you might use only a decorative oversized opening mark for a pull quote, which is separate from the normal flow of text:

blockquote {
  display: flex;
  flex-direction: column;
  align-items: center;
}

blockquote::after {
  content: "";
  display: block;
  width: 80px;
  height: 5px;
  background-color: #98838e;
}

Quote marks library

Block quotes don‚Äôt necessarily need quote marks, but when you use them with purpose, they become more than punctuation. They become part of the design personality. Decorative marks are ideal when a brand wants to infuse its character into a design.

Sadly, even the nicest designed typefaces can include dull and uninspiring quote marks. So, it‚Äôs important to remember that you can choose marks from an altogether different font if they better suit a design.

That‚Äôs why, whenever I audition a new typeface, I check its quote marks. If they‚Äôre memorable or noteworthy, I add them as SVGs to my quote marks library so I can easily find them later.

Shapes

Quotation design needn‚Äôt stop at borders and quote marks. Block and pull quotes can be any shape. Or, you could use a clip-path or mask to transform a quotation into any shape you can imagine.
