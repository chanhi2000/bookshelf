42. lsb_release ‚Äì Check Your Distro Info

The lsb_release command displays Linux distribution-specific information such as version, ID, release, and codename.

If lsb_release is not installed, you can use apt to install lsb-core on Debian, or yum to install redhat-lsb on Red Hat.

,,

,,,
sudo dnf install redhat-lsb

Note

The -a option displays all available distribution information, including version, ID, description, release, and codename.

43. nc -zv localhost 80 ‚Äì Check If a Port Is Open

The nc command (short for netcat), which is used to check if a particular port on your computer is open and accepting connections. It‚Äôs a very handy tool for troubleshooting network or server issues.

Checking port 80 (used by web servers):

If port 80 is open and a service is running (like Apache or Nginx), you‚Äôll see something like:

Checking port 8080 (another common web port):

If there‚Äôs no service running on that port, you‚Äôll see:

44. curl ipinfo.io ‚Äì Find Your Public IP & Location Info

This command is a quick and easy way to check your system‚Äôs public IP address and some basic location info based on that IP.

When you run it, it sends a request to the website ipinfo.io, which replies with a bunch of useful details about your connection, such as:

  • The hostname (if available)
  • The country, region, and sometimes the city
  • The geolocation (latitude and longitude)
  • The ISP or organization you‚Äôre connected to

45. Find Files Owned by a Specific User in a Directory

The find command helps you find all the files in the current directory (and its subdirectories) that are owned by a specific user ‚Äì in this case, the root user.

This will list every file in the current folder and below that is owned by the user root.

46. sudo apt build-dep ‚Äì Automatically Install Build Dependencies

The sudo apt build-dep command automatically installs all the required development packages (called dependencies) that are needed to compile a specific software from source.

Let‚Äôs take the example below:

This command will look at what‚Äôs needed to build the ffmpeg package from source, then download and install all those libraries and tools for you. It saves you from having to figure out and install each required package manually ‚Äì which can be confusing and time-consuming.

So instead of getting stuck with error messages like ‚Äúmissing dependency‚Äù during compilation, this command sets up everything for you in advance.

47. lsof -iTCP:80 -sTCP:LISTEN ‚Äì Check What‚Äôs Running on a Port

This command helps you find out which process or service is using a specific TCP port, like port 80 (used by web servers).

Let‚Äôs break down the command explanation:

  • lsof = ‚ÄúList Open Files‚Äù (in Linux, everything is treated like a file ‚Äì even network connections).
  • -iTCP:80 = Filters the output to show only TCP connections on port 80.
  • -sTCP:LISTEN = Shows only services that are actively listening for connections on that port.

48. find -size +100M ‚Äì Find Large Files Easily

Sometimes your disk gets full, and you wonder what‚Äôs eating up all that space? That‚Äôs where the find command comes in super handy and helps you search for big files (above a certain size) starting from your current directory and checking every folder inside it ‚Äì that‚Äôs what we call ‚Äúrecursively‚Äù.

Here‚Äôs what it does:

  • . ‚Äì means start looking from the current directory.
  • -size +100M ‚Äì means look for files larger than 100 megabytes.

If you want, you can find even bigger files, such as 1000 MB (that‚Äôs 1 GB):

This will only find really large files that are over 1 GB in size.

49. pdftk ‚Äì Merge Multiple PDF Files into One

The pdftk (PDF Toolkit) command is a handy tool that lets you combine multiple PDF files into a single file, which is super useful when you have several documents, such as reports, scanned pages, or ebooks, and want to keep them all together in one neat file.

You‚Äôll need to have pdftk installed on your system.

50. ps -LF -u ‚Äì Check User Processes and Threads

The following command shows all the running processes and threads for a specific user in a detailed format.

ps -LF -u ravi

Let‚Äôs break it down:

  • ps is the standard command to list running processes on your system.
  • -L shows threads (not just processes), which is helpful because one process can have multiple threads.
  • -F gives you a full-format listing, meaning you‚Äôll see more columns with detailed information.
  • -u user_name limits the results to processes run by a specific user (replace user_name with the actual username).

This will display all processes and threads run by the user ravi, including Process IDs, Thread IDs, CPU usage, Start time, and the actual command or program being run.

51. Startx ‚Äî :1 ‚Äì Run Multiple Graphical Sessions

Ever found yourself needing to log in and out of your graphical desktop just to switch users or run a separate session? That‚Äôs where the startx command comes in handy.

The startx -- :1 command lets you start a brand-new X session (a separate graphical desktop environment) without closing your current one, which means you can have multiple GUI sessions running at the same time ‚Äì useful for testing, multitasking, or switching between users without logging out.

Once you‚Äôve started a second X session using:

You can switch between the sessions using keyboard shortcuts:

  • Press Ctrl+Alt+F7 to go to your original X session.
  • Press Ctrl+Alt+F8 to switch to the new session you just started.

Most Linux systems reserve Ctrl+Alt+F1 to F6 for non-graphical console terminals, and F7 to F12 for graphical X sessions. So technically, you can run multiple GUI environments and switch between them using these shortcuts.
